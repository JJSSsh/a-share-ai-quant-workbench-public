# Collaboration

这个仓库的目的不是招生，也不是做培训，而是寻找少数能一起研究 A 股 AI 量化工作台的人。

## 我想找什么人

- 会 Python / pandas / SQL / DuckDB 的工程型研究者。
- 熟悉 A 股盘口、短线、尾盘、涨停机制的人。
- 能做复盘、归因、数据标注的人。
- 对 QMT / MiniQMT / Streamlit / 本地数据库有经验的人。
- 能提出策略假设，并愿意接受样本验证的人。
- 能把观点落到代码、表结构、样本或复盘记录里的人。

## 我不找什么人

- 想要现成股票代码的人。
- 想跟单、带单、荐股的人。
- 只聊概念但不做验证的人。
- 只想白嫖完整代码和策略参数的人。
- 把 AI 当玄学预测器的人。
- 不能接受真实数据和私有策略不公开的人。

## 私信前可以先回答

1. 你现在主要研究 A 股的哪类策略？短线、趋势、打板、尾盘、量化、基本面都可以。
2. 你会哪些工具？Python / SQL / pandas / DuckDB / QMT / Streamlit / Linux / Docker。
3. 你过去有没有做过回测、复盘或实盘观察？可以简单说一个例子。
4. 你更想贡献哪一块：代码、数据、复盘、策略假设、新闻归因、盘口观察。
5. 你能接受不荐股、不带单、不公开私有策略，只做验证共研吗？

## 协作原则

- 先小范围交流，再决定是否进入共研。
- 以具体问题为入口，不建泛泛闲聊大群。
- 优先讨论可验证假设、数据结构、复盘方法和工程实现。
- 尊重每个人的私有数据、账户、策略参数和交易边界。
