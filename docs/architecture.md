# Architecture

公开版只描述研究工作台的核心闭环，不包含真实数据路径和私有策略实现。

## 核心闭环

```text
行情与事件
  -> 候选信号池
  -> 策略指标与过滤
  -> 观察池 / 持仓观察
  -> 盘后复盘
  -> 归因与策略修正
  -> 下一轮样本验证
```

## 分层

### 1. 数据层

负责读取行情、竞价、日线、分钟线、资金流、新闻事件等数据。公开版只提供 mock 数据和 schema，不提供真实数据库。

### 2. 信号层

负责把候选股票、触发原因、策略标签、风险标记和观察窗口落入统一表。核心目标是让信号可追踪，而不是只停留在临时 CSV。

### 3. 观察层

负责记录候选信号后续表现，包括是否进入观察池、是否触发买入条件、是否进入持仓观察、何时失效。

### 4. 复盘层

负责盘后归因，包括成功样本、失败样本、未触发样本、误报样本和错过样本。AI 主要参与总结、分类和提出待验证假设。

### 5. 展示层

本地页面用于查看策略卡片、信号池、观察池、复盘记录和指标变化。公开版不承诺提供完整 UI。

## AI 的位置

AI 不作为玄学预测器，也不直接给出买卖建议。它更适合承担：

- 把新闻、公告、题材和股票映射为结构化候选。
- 辅助生成复盘摘要。
- 对失败样本做归因分类。
- 检查策略规则是否前后矛盾。
- 帮助生成 SQL、测试样例和研究笔记。
